CSS selectors

CSS selectors are how you get things on the page to style them in your CSS. There are a few basic ways to do this:

  • by element: a {color: red;}
  • by class: .danger {color: red;}
  • by id: #warning-message {color: red;}

When it comes to ids and classes, remember:

  • An id should only appear once on the page. Ids should be unique and elements can only have one id.
  • Classes may be applied to many elements on the page.
  • Elements can have many classes!

Select on multiple things on a single element by chaining selectors together without a space, for example: a.danger.btn {color: red;}

Descendant CSS selectors

You can select nested elements by putting a space between selectors. For example: header a selects all the <a> tags in the header.

Some rules of thumb when nesting CSS selectors:

  • Keep it shallow! Deeply nested selectors can get confusing.
  • Omit any unnecessary selectors. Keep it as simple as possible!

Pseudo-selectors!

Pseudo-selectors allow you to create CSS selectors that select elements based on a particular state of that element (called pseudo-classes). They are denoted by a colon. For example: a:hover selects the <a> tags that are currently being hovered over by the cursor.

A list of all the available pseudo-classes is available on MDN.

Some rules of thumb when using pseudo-selectors:

  • Use them sparingly! If you're uncertain, you probably don't need it.
  • Don't get too clever -- It's really easy for pseudo-selectors to get confusing!

CSS Layouts

The way we've managed CSS layouts has evolved over the years. In chronological order:

  • Tables: using the <table> tag, you can create columns and rows. This (sort of) allows you to manage complex layouts, but it's really clunky and has some limitations
  • Floats: using the CSS property float:left; or float:right; pulls elements out of the layout flow. This has been used to create column layouts, but floating has some idiosyncrasies that require proactive management. Floated elements must be "cleared", and this can cause problems.
  • Flexbox: using the CSS property display:flex; is the modern way to manage complex layouts with CSS. It allows you to flexibly size child elements using some special CSS properties such as flex. You can find more info at MDN.

There are 3 ways to set child width when using display:flex;:

  • By default the element will be the width of the content inside it
  • Horizontal width in px using the width property
  • Proportional sizing using the flex property. Elements with a flex value will get a proportion of the entire remaining width of the parent element based on the ratio of it's flex value to those of all other child elements with a flex property
